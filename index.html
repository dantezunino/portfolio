Generic Board

Interactive multiuser tool




Same as the monster database, this one here is a tool to make the life of my friends and I easier at the moment of play the game. Playing a TTRPG online has some troubles, being one of them the one that the absense of a board produce, and describing the position of everything and everyone all the time is not fun. The solution was to make a board.

This project is pure functionallity, and only serves the purpose of let everyone knows where they are. To do it I used Socket.io, sending the coordenates and color of each square to the server and, from there, to all the online players. You can test this opening the board in 2 different tabs and seeing what happends when you click a square in one.

What does this project try to show?

  • Ability to develop live interactive pages
  • Knowledge of different tools, like Socket.io
  • Ability to read new documentation and understand the content
  • Practicity
Fake Social Network

Live interactions with JSON repo



I was learning to use Svelte and decided it was a good idea to use their framework (Sapper) to make something for this portfolio.

The idea was changing over the days and ended up being this. It's a posting API that saves that post in a JSON file in JSONbin.io, also shows you the last 2 post made. If we added a user login, authentication and sessions this could be a normal operation in a mainstream social network. It also shows you fake trending posts from jsonplaceholder.

In the other 2 tabs there is an About that tells you a little more about the page, and a silly game. The style of the page is possible thanks to Bulma.io.

What does this exercise try to show?

  • Knowledge of frameworks like Sapper
  • Understanding of JSON format and DB search
  • Knowledge in handling Promises and HTTP protocols
  • Use of tools like the ones Svelte provides to reduce code and better the UX

Tablero genérico

Herramienta interactiva multi-usuario



Al igual que con la base de datos de monstruos ésta herramienta está hecha para facilitarnos la vida a mí y a mis amigos. Jugar un TTRPG por internet tiene algunos problemas, siendo uno de ellos el hecho de carecer de un tablero, y describir la posición de cada cosa y cada jugador todos los turnos no es divertido. La solución, entonces, fue hacer un tablero.

Este proyecto es pura funcionalidad y sólo sirve el propósito de dejarle saber a cada uno dónde se encuentra. Para hacerlo usé Socket.io, enviando al servidor las coordenadas y color de cada cuadrado, y una vez desde allí a todos los jugadores. Usted mismo puede testearlo abriendo el tablero en 2 pestañas diferentes y viendo qué ocurre cuando clickea un cuadrado en alguna de ellas.

¿Qué intenta mostrar este proyecto?

  • La habilidad para desarrollar páginas interactivas en vivo
  • El conocimiento de distintas herramientas, como Socket.io
  • La habilidad para leer nueva documentación y comprender el contenido
  • Practicidad
Red Social Falsa

Interacciones en vivo con repo de JSON



Estaba aprendiendo a usar Svelte y decidí que era una buena idea usar su framework (Sapper) para hacer algo para este portfolio.

La idea fue cambiando a lo largo de los días y termino de esta forma. Es una API que guarda un post en un JSON, alojado en JSONbin.io. También te muestra los últimos dos posteos hechos. Si agregaramos un loggeo y autenticación de usuario esta sería una operación normal de una red social mainstream. También muestra un grupo de falsos posteos en tendencia de jsonplaceholder.

En las otras 2 pestañas hay un "About" que cuenta un poco más sobre la página, y también hay un juego bastante tonto. El diseño de la página hay que agradecérselo a Bulma.io.

¿Qué intenta mostrar este ejercicio?

  • Conocimiento de frameworks como Sapper
  • Entendimiento del formato JSON y de búsqueda en base de datos
  • Conocimiento en manejo de Promesas y protocolos HTTP
  • Uso de herramientas como las que provee Svelte para reducir el código y mejorar la experiencia de usuario
